Integration · Setup guide

Connect Follow Up Boss to LeadCove.

A 30-second connection. Paste your Follow Up Boss API key once, and every lead you enrich from that point forward auto-pushes into FUB — deduped by phone or email, tagged with score and source so your SmartLists can pick them up automatically.

🌊 → 📇
LeadCove preps the contacts. Follow Up Boss runs the conversation. Every newly enriched lead lands in your FUB account the instant owner data resolves — with the phone, email, score, and tags pre-filled.

Before you start

You'll need an active Follow Up Boss account (any tier) and a LeadCove account on the Pro, Power, or House plan. If you're on Starter, upgrade in Settings → Plan first — the integration unlocks the moment your subscription updates.

Total time, start to finish: about 30 seconds, plus whatever it takes you to log into both tabs.

Setup — three steps

Get your Follow Up Boss API key

In Follow Up Boss, go to Admin → API → Create API Key. Name it "LeadCove" so future-you remembers what it's for, and copy the key to your clipboard.

Direct link: app.followupboss.com/api

Paste it into LeadCove

In LeadCove, click ⚙ Settings in the bottom-left of the sidebar, scroll down to the 📇 Follow Up Boss section, and paste your API key. Click Connect FUB.

We test the credential against your FUB account before saving. Invalid keys get rejected with a clear error message — you won't end up with a half-connected state that fails silently later.

You're done. Try it.

Drop a CSV into LeadCove (or click Enrich on a single lead). The instant owner data resolves, that contact appears in your Follow Up Boss account — deduped by phone/email if FUB already had a similar record, otherwise created fresh.

Each pushed contact carries tags so you can SmartList in FUB the same way you filter in LeadCove: source, lead-source, list name, score tier, score, property type, equity percentage, and years held.

Verifying the connection worked: after your first enrichment, refresh your Follow Up Boss People view. The new contact will be there, tagged with leadcove at minimum (plus all the LeadCove-specific tags). If you don't see it within 30 seconds, jump to the troubleshooting section below.

What gets synced

The integration pushes new enrichments forward from the moment you connect. Existing data in either tool stays right where it is — we don't migrate, overwrite, or modify your historical FUB contacts.

What gets pushed on each enrichment

  • Owner first + last name — including the real person behind LLC / Trust / Corp ownership.
  • Primary phone — mobile preferred, landline as fallback.
  • Email address — best available match.
  • Property address — full street + city + state + ZIP.
  • Joint-owner data — when a property has two named owners (spouses, business partners), both contacts go in linked to the same address.
  • Tags — leadcove, source:<label>, lead-source:<raw>, list:<name>, tier:<A|B|C|Hold>, score:<N>, type:<property type>, equity:<N>%, held:<N>y.

What we don't push

  • Rows we couldn't enrich. If a lead didn't return usable contact data, it stays in LeadCove only — no "Property Owner @ 123 Main St" placeholders polluting your FUB account.
  • Excluded rows. PO boxes, generic mail-forwarding addresses, government-owned parcels — filtered out before they reach your dashboard, let alone FUB.
  • DNC / TCPA-flagged contacts. Surfaced inside LeadCove but never pushed to FUB — you decide how to handle them.

Manual sync — for the leads you already enriched

Auto-sync only fires on new enrichments after you connect. For leads already enriched in LeadCove (before you wired up FUB), use one of three manual sync surfaces:

  • Per-list — click the 📇 icon next to any list in your sidebar to push every enriched lead in that list to FUB.
  • All Leads — the 📇 Sync to FUB button in the All Leads page header pushes everything.
  • Selected leads — check the boxes on specific rows, then use the 📇 Sync to FUB button in the bulk-action bar.

All three surfaces are idempotent — already-synced rows are skipped automatically (we know because each row gets stamped with its FUB contact ID after the first push). You can re-run a sync as many times as you want without creating duplicate FUB contacts.

Troubleshooting

I pasted my key but the connection failed.

Double-check that you copied the full key from FUB — they're long, and a partial paste won't validate. The Settings drawer will show the validation error verbatim, which tells you whether the key is malformed, expired, or rejected by FUB's API.

If you regenerate the key in FUB, paste the new one and click Update — we replace the old credential and the integration picks up immediately.

I enriched a lead but don't see it in FUB.

A few things to check, in order: (1) was the enrichment successful — i.e. did LeadCove actually find a phone or email for the owner? Auto-sync only fires when there's contact data to push. (2) Is your FUB filter view showing all contacts? Sometimes a SmartList narrows the view past where the new contact would appear. (3) Search FUB by the property address — the new contact may have been deduped into an existing FUB person record.

If none of those explain it, refresh the LeadCove dashboard and check Settings → Follow Up Boss — the Last sync timestamp tells you when the most recent push happened.

Will this create duplicate contacts in my FUB account?

No. The integration uses FUB's deduplication (by phone and email) on every push. If we find a contact for a property whose owner is already in your FUB account, we update that existing record instead of creating a duplicate.

We also stamp the LeadCove row with the FUB contact ID after each push, so re-running a sync is a no-op for already-synced rows.

Can I disconnect later?

Yes. Settings → Follow Up Boss → Disconnect removes the API key from LeadCove. Your existing FUB contacts stay right where they are — we never touch them on disconnect. Only new enrichments stop auto-pushing.

You can reconnect any time by pasting the key again.

Does my FUB account need a specific plan?

No. The integration works on any FUB plan that exposes the API (which is all of them). The plan-tier requirement is on the LeadCove side: you need to be on Pro, Power, or House to connect.

Is the API key stored securely?

Yes. Keys are encrypted at rest in our database, never logged in plaintext, and only decrypted in-memory when a sync fires. We never send your key to any third party other than the FUB API itself.
